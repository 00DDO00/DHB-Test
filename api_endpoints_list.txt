NOTES
==========================================

1. The application uses a hybrid approach with both YAML-compliant endpoints
   (primary) and legacy endpoints (for backward compatibility).

2. All endpoints require standard authentication headers.

3. Most endpoints return mock data for development purposes.

4. The YAML endpoints follow RESTful conventions and are the preferred
   endpoints for new development.

5. Legacy endpoints are maintained for existing functionality that hasn't
   been migrated to YAML-compliant endpoints yet.

6. Customer ID is typically hardcoded as 'CUST001' for development.

7. All endpoints include proper error handling and validation.

8. The base URL is http://localhost:5003 for the Java Spring Boot backend.
